### 关于"请求失败"的说明

经核查，所有失败请求均为 **ID 不存在**（已删除文档），而非文件缺失：

- 数据库文档总数: 2393
- ID 范围: 3 - 2613
- ID 空洞数: 218 个（约 8.3%）

测试脚本使用随机 ID (100-2600)，命中空洞时返回 404，这是正常行为。
实际存在的文档 ID 测试通过率 100%。

## 针对当前硬件的优化建议

### 硬件限制

当前服务器仅有 **1 核 CPU** 和 **894MB 内存**，这是典型的入门级云服务器配置。

### 建议

1. **不建议多 worker**: 单核环境下多进程反而增加上下文切换开销

2. **异步化改造**: 当前同步数据库操作会阻塞事件循环
   - 将 `document_db.py` 改用 `aiosqlite`
   - 将同步端点改为 `async def`

3. **减少内存占用**:
   - ZIP 文件流式读取而非全量加载
   - 限制并发请求数 (nginx `limit_conn`)

4. **预生成缩略图**: 避免请求时实时生成

5. **合理预期**: 单核 894MB 内存下，5 并发是合理上限
